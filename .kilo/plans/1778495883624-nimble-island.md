## Goal
Ensure that orders placed on the Vercel production deployment are saved to MongoDB instead of in-memory storage, and push the updated code to GitHub.

## Current State
- The server checks for `MONGODB_URI` environment variable; if set, saves orders to MongoDB, otherwise uses in-memory storage (not persistent).
- Orders placed on Vercel production currently use in-memory due to missing `MONGODB_URI`.
- MongoDB is installed as a dependency, but no database is configured yet.

## Steps to Implement
1. **Set up MongoDB Atlas**:
   - Go to https://www.mongodb.com/atlas and create a free account.
   - Create a new project and cluster (choose free tier).
   - In Network Access, add IP address 0.0.0.0/0 to allow all connections.
   - In Database Access, create a user with password.
   - Get the connection string from Connect > Connect your application: mongodb+srv://<username>:<password>@<cluster-url>/restaurant_db?retryWrites=true&w=majority

2. **Configure environment variable locally**: Update the `.env` file in the `server` directory with `MONGODB_URI=mongodb://localhost:27017/restaurant_db` for local testing. Optionally, use the Atlas URI to test cloud connection locally.

3. **Test locally**: Start the server with `npm start`, place an order via Postman using the example JSON, and verify in MongoDB Compass that the order is saved. (Confirmed working)

4. **Set environment variable on Vercel**:

   - Redeploy the project to apply the variable.

5. **Test production**: Place an order on https://items7-98zr.vercel.app/cart and confirm it appears in MongoDB Atlas or Compass.

6. **Redeploy and troubleshoot**:

7. **Push to GitHub**:
