# Plan to Set Up MongoDB for Order Persistence

## Goal
Ensure that orders placed on the Vercel production deployment are saved to MongoDB instead of in-memory storage.

## Current State
- The server checks for `MONGODB_URI` environment variable; if set, saves orders to MongoDB, otherwise uses in-memory storage (not persistent).
- Orders placed on Vercel production currently use in-memory due to missing `MONGODB_URI`.
- MongoDB is installed as a dependency, but no database is configured yet.

## Steps to Implement
1. **Set up a MongoDB database**: Create a free account on MongoDB Atlas (https://www.mongodb.com/atlas), create a cluster, and get the connection string (replace <username>, <password>, <cluster-url>).
2. **Configure environment variable locally**: Update the `.env` file in the `server` directory with `MONGODB_URI=mongodb://localhost:27017/restaurant_db` for local testing.
3. **Test locally**: Start the server with `npm start`, place an order via the frontend or Postman, and verify in MongoDB Atlas dashboard that the order is saved.
4. **Set environment variable on Vercel**: In Vercel dashboard, go to project settings > Environment Variables, add `MONGODB_URI` with the same connection string.
5. **Redeploy on Vercel**: Once rate limit is lifted, push changes or redeploy to apply the env var.
6. **Test production**: Place an order on the Vercel link and confirm it appears in MongoDB Atlas.

## Considerations
- Ensure the MongoDB cluster allows connections from Vercel's IP ranges (or use 0.0.0.0/0 for simplicity, but secure in production).
- The connection string includes database name `restaurant_db` as per server code.
- If using a different MongoDB provider, adjust the URI accordingly.
