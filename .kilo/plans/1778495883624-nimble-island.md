# Plan to Set Up MongoDB for Order Persistence

## Goal
Ensure that orders placed on the Vercel production deployment are saved to MongoDB instead of in-memory storage.

## Current State
- The server checks for `MONGODB_URI` environment variable; if set, saves orders to MongoDB, otherwise uses in-memory storage (not persistent).
- Orders placed on Vercel production currently use in-memory due to missing `MONGODB_URI`.
- MongoDB is installed as a dependency, but no database is configured yet.

## Steps to Implement
1. **Set up MongoDB Atlas**:
   - Go to https://www.mongodb.com/atlas and create a free account.
   - Create a new project and cluster (choose free tier).
   - In Network Access, add IP address 0.0.0.0/0 to allow all connections.
   - In Database Access, create a user with password.
   - Get the connection string from Connect > Connect your application: mongodb+srv://<username>:<password>@<cluster-url>/restaurant_db?retryWrites=true&w=majority

2. **Configure environment variable locally**: Update the `.env` file in the `server` directory with `MONGODB_URI=mongodb://localhost:27017/restaurant_db` for local testing.

3. **Test locally**: Start the server with `npm start`, place an order via Postman using the example JSON, and verify in MongoDB Compass that the order is saved. (Confirmed working)

4. **Set environment variable on Vercel**:
   - Go to https://vercel.com/dashboard/projects/items7-98zr/settings/environment-variables
   - Add a new variable: Name `MONGODB_URI`, Value: mongodb://amandahend20_db_user:<your_password>@ac-z9xofev-shard-00-00.ex8b11n.mongodb.net:27017,ac-z9xofev-shard-00-01.ex8b11n.mongodb.net:27017,ac-z9xofev-shard-00-02.ex8b11n.mongodb.net:27017/?ssl=true&replicaSet=atlas-eynuub-shard-0&authSource=admin&appName=Cluster0 (replace <your_password> with your Atlas user password).
   - Redeploy the project to apply the variable.

5. **Test production**: Place an order on https://items7-98zr.vercel.app/cart and confirm it appears in MongoDB Atlas or Compass.

6. **Troubleshoot if not saving**:
   - Check Vercel deployment logs: Go to Vercel dashboard > Deployments > Click on latest deployment > View Logs.
   - Look for MongoDB connection errors.
   - Ensure the Atlas cluster is active and the URI is correct.

## Considerations
- Ensure the MongoDB cluster allows connections from Vercel's IP ranges (or use 0.0.0.0/0 for simplicity, but secure in production).
- The connection string includes database name `restaurant_db` as per server code.
- If using a different MongoDB provider, adjust the URI accordingly.
